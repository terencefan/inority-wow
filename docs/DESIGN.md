# docs Design

## Purpose

This folder holds hand-maintained deep-dive documentation that is more specific than the root README and more durable than generated tool output.

## Current Contents

- `LootPanel.md`: detailed explanation of the loot panel surface and behavior.

## Design Constraints

- Use `docs/` for topic-driven documentation, not folder indexes.
- Keep `docs/` synchronized with the code when major UI or workflow semantics change.
- Prefer linking back to source-owned `DESIGN.md` files rather than duplicating architecture inventories in multiple places.
